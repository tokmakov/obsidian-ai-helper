# AI Helper — документация для разработчиков

## Требования

- Node.js 18+
- npm 9+

## Сборка

```bash
git clone https://github.com/evgeniy/obsidian-ai-helper
cd obsidian-ai-helper
npm install

# Режим разработки с hot reload
npm run dev

# Продакшн сборка
npm run build
```

Результат сборки — файлы `main.js` и `styles.css` в корне проекта.

## Архитектура

```
src/
├── main.ts            # точка входа, инициализация плагина
├── types.ts           # типы и настройки по умолчанию
├── ai-client.ts       # HTTP клиент для работы с API
├── chat-view.ts       # UI панели чата
├── settings.ts        # страница настроек плагина
├── session-manager.ts # управление историей сессий
└── logger.ts          # логирование запросов и ответов
```

## Описание модулей

### main.ts
Точка входа плагина. Отвечает за:
- Инициализацию всех модулей (`Logger`, `SessionManager`, `AIClient`)
- Регистрацию боковой панели и команд
- Очистку старых логов и сессий при старте
- Применение настроек шрифта

### types.ts
Общие типы данных:
- `AIPluginSettings` — все настройки плагина с дефолтными значениями
- `ChatMessage` — одно сообщение в чате (роль, текст, время)

### ai-client.ts
HTTP клиент для работы с OpenAI-совместимым API. Отвечает за:
- Отправку запросов к API провайдера
- Retry-логику при сетевых ошибках (до 5 попыток с паузой 1 сек)
- Парсинг ответа и извлечение подсказок из фигурных скобок
- Получение списка доступных моделей

**Retry-логика** — при `ERR_CONNECTION_RESET` или других сетевых ошибках клиент автоматически повторяет запрос. Это решает проблему нестабильных провайдеров.

**Парсинг подсказок** — ассистент может добавлять в конец ответа блок с вопросами в формате `{Вопрос 1}` `{Вопрос 2}`. Клиент извлекает их и передаёт в UI отдельно от основного текста.

### chat-view.ts
UI боковой панели, отвечает за
- Отрисовку сообщений с поддержкой Markdown
- Управление сессиями (создание, переключение, удаление)
- Автосохранение сессии после каждого ответа
- Кнопки действий под сообщениями (копировать, вставить в заметку)
- Отображение подсказок для следующего вопроса

### settings.ts
Страница настроек в Obsidian, параметры

| Поле                    | Тип     | Описание                        |
|-------------------------|-------- |---------------------------------|
| `apiKey`                | string  | API ключ провайдера             |
| `baseUrl`               | string  | Base URL провайдера             |
| `model`                 | string  | Выбранная модель                |
| `availableModels`       | string  | Список доступных моделей        |
| `fontSize`              | number  | Размер шрифта панели (10–24)    |
| `loggingEnabled`        | boolean | Включить логирование            |
| `logsRetentionDays`     | number  | Хранить логи N дней (10–100)    |
| `sessionsRetentionDays` | number  | Хранить сессии N дней (10–1000) |

### session-manager.ts
Управление историей сессий. Сессии хранятся как JSON файлы:

```json
{
    "id": "2026-03-26T11-47-43-000Z",
    "title": "Как установить git под Ubuntu",
    "createdAt": "2026-03-26T11:47:43.774Z",
    "updatedAt": "2026-03-26T11:52:10.877Z",
    "messages":
}
```

Название сессии формируется автоматически из первых 75 символов первого вопроса пользователя, обрезается по последнему слову.

### logger.ts
Логирование запросов и ответов в файлы. Особенности:
- Использует `vault.adapter` напрямую — Obsidian не индексирует папку `.obsidian`, поэтому стандартный Vault API (`getAbstractFileByPath`) не работает для файлов внутри неё
- Один файл лога в день: `YYYY-MM-DD.log`
- Автоматическая очистка файлов старше N дней

## Добавление нового провайдера

Плагин работает с любым OpenAI-совместимым API. Для подключения нового провайдера достаточно изменить два поля в настройках:

1. **Base URL** — адрес API, например `https://api.new-provider.com/v1`
2. **API Key** — ключ доступа от провайдера

## Известные особенности

**Vault API не работает для `.obsidian/`**
Obsidian исключает папку `.obsidian` из индексации, поэтому `vault.getAbstractFileByPath()` возвращает `null` для файлов внутри неё. Решение — использовать `vault.adapter` напрямую (методы `read`, `write`, `exists`, `mkdir`, `list`, `remove`).

**Подсказки и фигурные скобки в ответах**
Если ответ ассистента содержит фигурные скобки (шаблоны, конфиги), парсер подсказок ищет только последний блок из 2–3 `{...}` идущих подряд в конце ответа — остальные фигурные скобки игнорируются.
